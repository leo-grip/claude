# Settlement Project

## 프로젝트 개요

그립 커머스 플랫폼의 정산 및 지급 시스템. 

## 작업 규칙

### 빌드 체크
- 코드 수정 작업이 완료되면 반드시 `./gradlew build`를 실행하여 빌드가 되는지 확인한다.
- 빌드 실패 시 원인을 분석하고 수정한 뒤 다시 빌드를 확인한다.

### 컨텍스트 저장
- `/compact` 실행 전에 반드시 context-save 에이전트를 실행하여 `.claude/context-snapshots/`에 스냅샷을 남긴다
- 스냅샷은 사람이 읽을 수 있는 줄글 형태로 작성한다
- 어떤 작업을 왜 했는지, 현재 어디까지 진행되었는지, 남은 작업은 무엇인지를 자연스러운 문장으로 설명한다
- 코드 파일 목록이나 구조체 나열보다 작업의 흐름과 맥락을 전달하는 것을 우선한다

### 작업 진행 보고
- 작업이 길어질 경우, 약 1분 간격으로 현재 진행 상황을 사용자에게 보고한다.
- 보고 후 이어서 작업을 계속 진행한다.
- 보고를 할 때는 핵심만 간결하게 전달한다.

### 외부 서비스 MCP
- 모든 MCP는 **읽기 전용**으로만 사용한다.
- 이슈 생성/수정, 페이지 생성/수정, 댓글 작성 등 쓰기 기능은 절대 사용하지 않는다.
- 메시지 전송, 채널 생성, 리액션 추가 등 쓰기 기능은 절대 사용하지 않는다.
- Jira 검색 시 GRIPSVR(서버) 프로젝트에만 한정하지 않는다. 기획 등 다른 프로젝트의 티켓도 함께 검색하여 관련 컨텍스트를 폭넓게 파악한다.
